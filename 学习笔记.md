# LightNovelReader 插件开发学习笔记

## 一、项目概述

LightNovelReader（轻小说阅读器）是一款基于 Kotlin + Jetpack Compose 的开源 Android 应用，支持插件扩展。

### 主要技术栈
- **语言**: Kotlin 100%
- **UI 框架**: Jetpack Compose + Material3
- **最低版本**: Android 7.0 (API 24)
- **目标版本**: Android 15
- **构建工具**: Gradle (Kotlin DSL)

## 二、插件系统架构

### 2.2 插件入口点

必须满足以下条件：
1. 有且仅有一个实现 `LightNovelReaderPlugin` 接口的类
2. 为该类添加 `@Plugin` 注解

```kotlin
@Plugin(
    version = 1,
    name = "PluginName",
    versionName = "1.0.0",
    author = "Author",
    description = "Description",
    updateUrl = "https://example.com",
    apiVersion = 1
)
class MyPlugin : LightNovelReaderPlugin {
    override fun onLoad() {
        // 初始化代码
    }
}
```

### 2.3 依赖注入

通过构造器参数获取软件内部 API 实例：

```kotlin
class MyPlugin(
    private val userDataRepositoryApi: UserDataRepositoryApi,
    private val userDataDaoApi: UserDataDaoApi,
    private val bookRepositoryApi: BookRepositoryApi,
    private val bookshelfRepositoryApi: BookshelfRepositoryApi,
    private val localBookDataSourceApi: LocalBookDataSourceApi
) : LightNovelReaderPlugin() {
    override fun onLoad() {
        // 使用 API
    }
}
```

### 2.4 可用的 API

| API | 用途 |
|-----|------|
| UserDataRepositoryApi | 用户数据仓库，存储键值对数据 |
| UserDataDaoApi | 用户数据 DAO，访问底层存储 |
| BookRepositoryApi | 书籍仓库，管理书籍数据 |
| BookshelfRepositoryApi | 书架仓库，管理书架 |
| WebBookDataSourceManagerApi | 网络书籍数据源管理 |
| TextProcessingRepositoryApi | 文本处理仓库 |
| LocalBookDataSourceApi | 本地书籍数据源 |

### 2.5 UI 自定义

通过重写 `PageContent` Composable 函数在插件页面显示自定义内容：

```kotlin
@Composable
override fun PageContent(paddingValues: PaddingValues) {
    Column(
        modifier = Modifier
            .padding(paddingValues)
            .clip(RoundedCornerShape(16.dp)),
        verticalArrangement = Arrangement.spacedBy(2.dp)
    ) {
        // 使用 SettingsSwitchEntry, SettingsClickableEntry 等组件
    }
}
```

## 三、API 文档

文档地址: https://api-doc.lnr.nariko.org/

### 3.1 UserDataRepositoryApi

用于存储和读取用户数据（键值对）。

常用方法：
- `stringUserData(key: String): StringUserData` - 字符串数据
- `booleanUserData(key: String): BooleanUserData` - 布尔数据
- `intUserData(key: String): IntUserData` - 整数数据
- `longUserData(key: String): LongUserData` - 长整数数据

每个 UserData 对象支持：
- `set(value)` - 设置值
- `getFlowWithDefault(default)` - 获取 Flow 流

### 3.2 BookshelfRepositoryApi

用于管理书架数据。

### 3.3 BookRepositoryApi

用于管理书籍数据。

## 四、WebDAV 协议

WebDAV (Web-based Distributed Authoring and Versioning) 是 HTTP 协议的扩展，支持：
- MKCOL - 创建目录
- PROPFIND - 获取属性
- GET - 下载文件
- PUT - 上传文件
- DELETE - 删除文件

### 4.1 常用 WebDAV 服务器

- Nextcloud
- ownCloud
- Synology Drive
- 任意支持 WebDAV 的 NAS

## 五、构建配置

### 5.1 build.gradle.kts 关键配置
